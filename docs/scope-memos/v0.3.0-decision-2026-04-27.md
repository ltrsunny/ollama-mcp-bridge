# v0.3.0 architectural decision — 2026-04-27

**Status:** decision committed, scope memo pending Prior Art Review.

This document captures the *outcome* of a 4-round Gemini ↔ Copilot brainstorm
on what v0.3.0 should actually deliver. The full `feature-intake-rule.md` flow
(Prior Art Review → scope memo → Auditor pass → code) follows.

## Why the rethink happened

Post-v0.2.0 reality check: the bridge ships fine, all 6 tools work, 59 unit +
58 smoke checks green. But the user observed each Claude Code session still
exhausts in 2-3 turns, and the bridge was used in <5 % of v0.2.0 cycle work.
**The bridge solves problems that don't dominate the frontier-token budget.**

## Where the brainstorm landed

Two AIs (Gemini 3.1 Pro Preview and Copilot CLI xhigh + GitHub MCP) ran a
4-round structured debate without the frontier in the middle. Round outputs
preserved at `/private/tmp/.../tasks/{ba6cmxr6k,buxsxfr1e,blmjhdowz,bivd0owws}.output`
for traceability.

The two AIs **independently converged** on three high-priority interventions
(strong cross-model agreement signal):

### Adopted into v0.3.0 (Phase 1)

**#1 Async Job + Persistent Memory Bank**

New MCP tools:
- `enqueue-job(toolName, args)` → returns `{ job_id }` immediately (< 1 s, far
  under MCP wall)
- `wait_for_job(job_id, max_wait_ms = 50000)` → blocks up to 50 s, returns
  `still_running` if not done; client calls again. Avoids naive 5 s polling
  that would burn frontier turns.
- `read_job_result(job_id)` → returns the persisted summary
- Storage: `.memory/jobs/<id>.md` + JSON sidecar; TTL 7 d; GC on bridge start

Estimated effort: **5-7 days** (Gemini said 3-5, Copilot said 5-8 with prod
hardening — Copilot's call is more honest).

Addresses bottleneck: **first-pass reading of long external research** +
**Claude Code 60 s wall** simultaneously.

**#2 `.claude/commands/draft-from-diff` hook + bridge `diff-semantic-index` tool**

Flow:
- User-side hook in `.claude/commands/draft-from-diff` reads `git diff`
- Calls bridge tool `diff-semantic-index(diff_text)` → local model emits
  structured JSON: `{change_type, files_touched, key_decisions[],
  risk_callouts[]}`
- Frontier writes the actual commit message / scope-memo content from the
  JSON, not from the raw diff

Estimated effort: **2-3 days**.

Addresses bottleneck: **long commit messages, scope memos, doc writing** —
output side of frontier-token consumption.

**Critical Gemini modification accepted**: do NOT have local model write the
final commit message. Local model produces *structured semantic index* of the
diff; frontier polishes. Local 7 B is unreliable enough at the prose layer
that bad drafts cost more frontier tokens to rewrite than they save.

### Rejected

**#3 Local Orchestrator + Multi-Pass Critique Protocol** (Copilot proposal)

Multi-round local critique with 7 B parameter models amplifies hallucinated
issues into noisy "contentious points". Frontier then must verify the noise,
negating savings. Already empirically observed in the v0.2.0 cycle: cross-
model critique works as a single sharp pass against a specific concrete
problem (e.g., the 8 K-bucket bug), but multiplied rounds with weaker
local models would degrade signal.

### Deferred

**#3 Pre-Emptive Map-Reduce Filtering Tool** (Gemini proposal — `extract-relevant-claims`)

Solid concept (parameterized query + map-reduce → only matching paragraphs)
but lower priority than #1 / #2. Defer until Phase 1 ships and real usage
shows whether external research reading is still a major sink even with
the Memory Bank in place.

**llama.cpp backend swap** (was the prior v0.3.0 plan)

Backend choice does not directly reduce frontier-token consumption. Real
pain is upstream of which engine runs the local model. Move llama.cpp
swap to **v0.4.0**.

## Cross-cutting deep disagreement (resolved)

- **Gemini**: local models should be restricted to lossy compression and
  retrieval; frontier retains all reasoning, critique, and final-artifact
  authority.
- **Copilot**: local models can do drafting and multi-pass critique with
  proper guards (confidence scores, hybrid retrieval, two-stage flows).

**Resolution**: Gemini wins on critique (local models too noisy for that),
Copilot wins on drafting (with the structured-index modification above).
Hybrid: local does compression, retrieval, AND structured drafting; local
does NOT do critique or open-ended generation.
